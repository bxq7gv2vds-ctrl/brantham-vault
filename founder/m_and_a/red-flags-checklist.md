# 40 Red Flags M&A — Checklist de Détection

## FINANCIÈRE (10 items)

- [ ] Churn >5%/mois (sanguin qui fuit)
- [ ] Croissance ralentit depuis >3 mois
- [ ] Marge brute <70% (coûts de serveur élevés, pas scalable)
- [ ] NRR <100% (décroissance attendue)
- [ ] CAC > LTV/24 mois (client acquisition ruineuse)
- [ ] Dépenses opex > 50% du revenu (pas optimisé)
- [ ] Trésorerie runway < 12 mois (désespéré)
- [ ] Customers concentrés : top 3 = >40% du revenu (risque client)
- [ ] Pricing instable : réductions fréquentes > 20% (pas de prix power)
- [ ] Multiples offerts vs benchmarks : écart >50% (overvalued ou cacher un problème)

## PRODUIT & TECH (8 items)

- [ ] Code legacy = >50% de la base (remplacer coûteux)
- [ ] Dépendances non-maintenues (npm packages deprecated)
- [ ] Architecture monolithique + pas de roadmap microservices
- [ ] Tests automatisés <50% de couverture (risque de régression)
- [ ] Uptime <99% historiquement (fiabilité douteuse)
- [ ] Tech stack exotique = talent rare requis (fuite de équipe certaine)
- [ ] Pas de roadmap produit claire au-delà de 3 mois
- [ ] Features core dépendent d'une seule personne (bus factor = 1)

## ÉQUIPE & TALENT (8 items)

- [ ] CEO/fondateur fatigué ou démotivé (attention lors des appels)
- [ ] CTO va partir (non négociable avant close)
- [ ] Top 3 talents déjà en job hunt (allez voir LinkedIn)
- [ ] Churn équipe >15%/an historique (culture problème)
- [ ] Aucun leader de produit senior (roadmap en risque)
- [ ] Pas de structure RH/Finance (improvisation après close = problème)
- [ ] Remote-first = dispersé (intégration culturelle difficile)
- [ ] Salaires au-dessus du marché (ajustement douloureux)

## CLIENTS & MARCHÉ (6 items)

- [ ] 1-2 clients = >30% du revenu (risque existentiel)
- [ ] NPS <40 (satisfaction douteuse)
- [ ] Churn acceleration depuis 6 mois (produit décline)
- [ ] Referenceable customers <50% (vraie traction ?)
- [ ] Marché fragmenté, pas de clear leader (terrain miné)
- [ ] Réglementation changeante = risque compliance (coûts cachés)

## LÉGAL & CONTRAT (5 items)

- [ ] Clients peuvent partir sans pénalité (contrats faibles)
- [ ] IP murkée (fondateur vs société, externe contributors)
- [ ] Open source liabilities non managées (GPL complications)
- [ ] Litiges actuels ou menaçants (divulgués récemment ?)
- [ ] Contrats clés manquent de confidentialité clauses (données exposées)

## OPÉRATIONNEL (3 items)

- [ ] Process ventes pas documenté (tribal knowledge)
- [ ] Aucun CFO ou bookkeeper sérieux
- [ ] Infrastructure coûts non optimisés (AWS mal configuré = quick win)

---

## Scoring Rapide

**Critère chaque flag :**
- **Vert** (pas d'alerte) : Continue
- **Jaune** (alerte, mais gérable) : Discount 10-15%, plan de mitigation
- **Rouge** (deal-breaker) : Walk away

**Règle 80/20** :
- >8 flags jaunes = rediscuss pricing
- >3 flags rouges = walk away

---

## Exemples de Mitigation (Jaune → manageable)

| Flag | Mitigation |
|------|---|
| Churn 5% | Augmenter retention budget post-close, freeze prix |
| Tech legacy | Budget 3-6 mois refactor, dédier team |
| Fondateur fatigué | Garder 2 ans bonus + nouveaux rôles |
| Top client 35% | Plan de diversification revenue 12 mois |
| Open source GPL | Audit IP + budget legal, possiblement restructurer |

---

## Utilisation

1. **Pre-outreach** (2 min) : Scanner publiquement (G2, Crunchbase, LinkedIn)
2. **Post-LOI** (30 min) : Due diligence approfdie
3. **Pre-close** (1h) : Rediscuss pricing si >3 jaunes

**Prochaine action** : Appliquer à 1-2 targets actuels, noter les couleurs.
