# Checklist Audit Rapide — Préparer SaaS à la Vente (30 jours)

## **FINANCIAL (Jour 1-5)**

### Revenue & Churn
- [ ] **ARR actuel exact** : €XXX (pas estimé, avec proof)
- [ ] **Croissance annuelle** : +60% (YoY documentation)
- [ ] **Net Revenue Retention** : +5% (ou -5% = issue)
- [ ] **Churn mensuel** : <5% (segment par segment)
- [ ] **ARPU trend** : Up/flat/down (price increases impact ?)
- [ ] **MRR concentration** : Top 10 clients = X% revenue (risk si >50%)

### Cash Flow
- [ ] **Runway** : X mois (good if > 12)
- [ ] **Annual burn** : €0 ou profitable
- [ ] **CAC payback** : <12 mois (if spending on sales)
- [ ] **LTV/CAC ratio** : >3x (magic threshold)

### Profitability
- [ ] **EBITDA margin** : +/-X% (show path to 20%+)
- [ ] **Gross margin** : >60% (SaaS standard)
- [ ] **OpEx breakdown** : Sales / Product / Admin / ...
- [ ] **Debt/funding** : None, or note terms

---

## **LEGAL & CORPORATE (Jour 6-10)**

### IP & Ownership
- [ ] **Source code** : All rights owned 100% by company
- [ ] **No open-source violations** : Check GPL, AGPL usage
- [ ] **Domains** : Company owns .com / .fr
- [ ] **Trademarks** : Registered or at least used (TM symbol OK)
- [ ] **Patents** : None (usually bad in SaaS, slow to exit)

### Corporate Structure
- [ ] **Bylaws current** : Not from 10 years ago
- [ ] **Cap table clear** : Who owns how much (no surprises)
- [ ] **Option grants** : All vested or note accelerations
- [ ] **Founder shares** : Locked, no weird side deals

### Contracts
- [ ] **Customer agreements** : Standard (no crazy commitments)
- [ ] **Vendor agreements** : Transferable post-sale (AWS OK, custom bad)
- [ ] **Employee NDAs** : Signed (non-compete may be issue)
- [ ] **Board seat / advisor** : Any weird side agreements ?

---

## **PRODUCT & TECHNOLOGY (Jour 11-15)**

### Architecture
- [ ] **Tech stack documented** : Node/Python/Java/Ruby + DB
- [ ] **Cloud hosting** : AWS/Azure/GCP (portable, good)
- [ ] **No deprecated libraries** : Should be <2 years old
- [ ] **Code quality** : Can a new team understand it in 1 week ?
- [ ] **Test coverage** : >60% (at least unit tests)

### Data & Security
- [ ] **GDPR compliant** : Privacy policy, data deletion, consent
- [ ] **Data backup** : Regular, tested, located
- [ ] **Encryption** : Transit (TLS) + at-rest (if sensitive data)
- [ ] **Security audit** : At least self-assessed (no major holes)
- [ ] **Incident history** : Zero critical breaches in 3 years (or disclosed)

### Operations
- [ ] **Uptime SLA** : 99.9%+ (track monthly)
- [ ] **Deployment process** : CI/CD automated (not manual)
- [ ] **Disaster recovery plan** : Documented, tested
- [ ] **Dependencies on founder** : No one knows system X except founder

---

## **CUSTOMER (Jour 16-20)**

### Customer Health
- [ ] **Top 10 customer retention** : >95% (no recent departures)
- [ ] **NPS score** : >40 (or show survey proof)
- [ ] **Support burden** : <X% revenue in support costs
- [ ] **Customer interviews done** : Sample 5-10 customers (buyer will)

### Market Position
- [ ] **Competitive advantages** : 3 things you do better than Competitor X
- [ ] **Switching costs** : Why don't customers leave ? (high = good)
- [ ] **Expansion revenue** : Cross-sell / upsell working ? (+10% extra good)
- [ ] **Free trial → paid conversion** : >10% (or market accept lower)

---

## **TEAM (Jour 21-25)**

### Key Person Risk
- [ ] **Founder leaving post-sale** : Retention clause clear
- [ ] **CTO essential ?** : Can engineering run without him/her ?
- [ ] **Sales person** : Can grow without original sales lead ?
- [ ] **Org chart** : Who reports to whom, any backups ?

### Hiring Ability
- [ ] **Open positions** : List what you're hiring for
- [ ] **Offer letters** : Conditional on close (or signed already)
- [ ] **Salary transparency** : Know your payroll is low/market/high

---

## **OPERATIONS (Jour 26-30)**

### Documentation
- [ ] **Runbook** : How to run the company (1 page min)
- [ ] **Customer documentation** : Updated, clear
- [ ] **Internal wiki** : Processes documented
- [ ] **Financial statements** : 3Y audited (or at least compiled)

### Liabilities & Risks
- [ ] **Lawsuits pending** : None (or disclosed)
- [ ] **Warranty issues** : Known bugs ? Bugs in customers' integrations ?
- [ ] **Tax compliance** : No years where you underreported
- [ ] **Insurance** : D&O coverage (buyer cares)

### Red Flags to Disclose
- [ ] **Churn spike** : Q2 had X% above avg (explain why)
- [ ] **Customer complaints** : Any vocal detractors ? (YouTube, Reddit)
- [ ] **Key departures** : If engineer left 6M ago, explain replacement
- [ ] **Competitive threats** : New player in market X (but you're winning)

---
