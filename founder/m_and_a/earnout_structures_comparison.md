# Earnout Structures — Options & Trade-offs

| Structure | Formula | Risk (for you) | Buyer Motivation | Example |
|---|---|---|---|---|
| **Revenue-based** | If ARR > $X → earn $Y | HIGH (dependent on buyer sales) | Low admin | If ARR>$3M → +$1M |
| **EBITDA-based** | If EBITDA > $X → earn $Y | MEDIUM (you control costs) | Incentivizes profit | If EBITDA>$300k → +$1M |
| **Product milestone** | If ship Feature X → earn $Y | MEDIUM (tech execution) | Tech-focused | If API integrations ship → +$500k |
| **Customer KPI** | If NRR > 120% → earn $Y | LOW (just manage team) | Growth aligned | If NRR stays >120% → +$500k |
| **Subjective** | At buyer discretion | VERY HIGH (no protection) | ❌ AVOID | "If integration smooth" |

**Recommendation:** Revenue-based + EBITDA-based hybrid
```
Year 1: If ARR > $2.5M → $500k
Year 2: If EBITDA > $300k → $1.0M
Total upside: $1.5M (spread across 2 years)
```
