# Session 2026-03-18 — MiroFish v0.3: 1M Agents Performance

## Objectif

Optimiser le moteur distille pour atteindre 1M agents x 100 rounds < 3 min sur M5.

## Point de depart

- Modele distille entraine (83.9% accuracy, 4293 params, 17.2KB)
- MLX inference 1M: 0.117s
- Mais le moteur complet: 144s/round (target: 1.8s/round) — 80x trop lent

## Optimisations realisees (par ordre)

### 1. Elimination des AgentAction objects
**Avant**: `_distilled_agent_round()` creait 937K objets `AgentAction` par round dans un Python for-loop.
**Apres**: Return `int`, appliquer les effets directement sur `self.env` (mutation directe des deals).
**Gain**: Elimine l'allocation de 937K objets Python par round.

### 2. Aggregation par deal (numpy)
**Avant**: Creation de 900K objets `Bid` dans un Python loop.
**Apres**: `np.bincount` + `np.unique` pour agreger les actions par deal. Un seul `Bid` (le meilleur) + un placeholder par deal par round.
**Gain**: De 900K Bid objects a ~600 (2 par deal actif).

### 3. Fix accumulation de bids
**Bug**: Les placeholder bids s'accumulaient: 20 par deal par chunk par round → 200K bids par deal apres 100 rounds.
**Fix**: Agreger les bids across all chunks AVANT d'appliquer. 1 best bid + 1 placeholder par deal par round total.

### 4. Geo scoring vectorise
**Avant**: Python dict groupby sur `agent_geo_ids` (frozensets) — O(n) Python loop.
**Apres**: Pre-compute `unique_geos` list + `agent_geo_group` int array. `geo_group_scores` matrix (n_groups x n_deals) calculee une fois. `geo_scores = geo_group_scores[chunk_groups]` = pure numpy fancy indexing.
**Gain**: ~200ms/chunk → ~50ms/chunk.

### 5. Buyer-only amplification
**Bug**: 155,100 institutional agents (tribunal, banque, mandataire, conseiller) etaient clones lors du multiplier.
Chacun appelait `agent.decide()` avec Python loops sur 10K+ deals.
**Fix**: `_amplify_population()` ne clone que les BUYER_TYPES. Les institutional restent a leur count de base (~2K).
**Gain**: De 20s/round a 0.83s/round.

### 6. maintien_emploi_pct fix
**Bug**: `maintien_emploi_pct=0.0` sur les Bid objects → tribunal rejetait 100% des offres → 0 cessions.
**Fix**: Set `maintien_emploi_pct=0.5` par defaut.
**Resultat**: Taux de cession 9% (realiste).

## Resultats finaux

| Metric | Avant | Apres | Improvement |
|--------|-------|-------|-------------|
| Temps/round (1M agents) | 144s | 0.83s | **173x** |
| Total 100 rounds | >4h | 83.4s | **~170x** |
| Agents distilled | 995,534 | 995,534 | - |
| Taux cession | 0% | 9% | Fixed |
| Baseline regression | 3.3s/50r | 3.3s/50r | No regression |
