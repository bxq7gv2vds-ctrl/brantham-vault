# Session 2026-03-16 — Brantham Simulator v2 Quant Research Lab

## What was done
Complete rewrite of Brantham Simulator from "startup dashboard" to "quant research lab" (Bloomberg terminal style).

### Backend (FastAPI, port 5001)
- **monte_carlo.py** — Monte Carlo engine (numpy, 1000 sims, calibrated distributions)
- **bayesian.py** — Bayesian network with CPTs for deal probability updates
- **metrics.py** — Deal metrics, market metrics (IBSE), probability matrix
- **models.py** — Added 8 quant models (MonteCarloResult, BayesianState, DealMetrics, AgentVector, MarketMetrics, QuantRoundSummary, QuantSimulationState)
- **agent_generator.py** — Pure math quant agents (conviction, bid_multiple, risk_score, sector_fit)
- **main.py** — 8 new API endpoints, 24 total routes
- Zero LLM required for quant path

### Frontend (React 19 + Vite, port 3333)
- Dark theme (#09090B), DM Mono font, dense CSS grid 3x2 layout
- All panels visible simultaneously (no tabs)
- Components: MetricsBar, GraphPanel, ProbabilityMatrix, MonteCarloChart, AgentGrid, SimulationControl, EventInjector, NodeMetrics, Telemetry, Sparkline
- Deleted old files: NodeDetail, AgentCards, SimulationPanel, GodsEyeView, SystemLog

## Key decisions
- Pure math for agent decisions (score = w1*conviction + w2*sector_fit + w3*deal_appetite - w4*risk_score)
- MC distribution stored as 101 percentiles (not full 1000 samples) for JSON efficiency
- Backward compatibility preserved for v1 LLM-based simulation endpoints
