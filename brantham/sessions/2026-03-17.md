# Session 2026-03-17 — MiroFish LLM Agents Activation

## Objectif
Activer les agents LLM (Qwen 2.5:7b via MLX) dans le world simulator MiroFish avec optimisations batch+sparse.

## Resultats
- **8 etapes du plan implementees et verifiees**
- 0 failures LLM apres fixes
- 5 agents LLM actifs sur 6884 total (ratio 0.1%)
- 80 actions LLM generees sur 20 rounds (~50s total)
- Dashboard, timeline (avec filtre LLM), graph tous fonctionnels

## Fichiers modifies (backend)
- `backend/ma/models/config.py` — +3 champs LLM (llm_call_interval, llm_batch_size, llm_max_tokens)
- `backend/ma/api/schemas.py` — +llm_ratio sur SimulationRequest, CompareRequest, MonteCarloRequest
- `backend/ma/api/router.py` — llm_ratio partout, stats LLM, filtre timeline agent_filter
- `backend/ma/agents/llm_agent.py` — batch_llm_decide(), stats tracking, regex JSON parsing
- `backend/ma/engine/simulation.py` — sparse rounds, batch, replay cached decisions
- `backend/ma/engine/monte_carlo.py` — accepte llm_ratio

## Fichiers modifies (frontend)
- `frontend/src/api/mirofish.js` — timeout 300s, agent_filter param
- `frontend/src/views/Terminal.vue` — panel LLM sidebar (toggle, slider, warning), ticker badge
- `frontend/src/components/PaneView.vue` — dashboard LLM KPIs, timeline filtre LLM, cyan styling
- `frontend/src/components/AgentGraph.vue` — noeuds LLM plus gros et cyan

## Bugs resolus
1. **JSON truncation** — LLM output tronque a 150 tokens. Fix: max_tokens=250 + regex `\{[^{}]+\}` pour parser objets individuels
2. **Agent ID/name mismatch** — LLM retourne agent.name (avec LLM-) mais map keyed par agent.id. Fix: agent_name_to_id mapping
3. **round_num=0** — Actions LLM sans round_num. Fix: set dans _batch_llm_round
4. **Replay round_num** — Decisions cachees gardaient le round original. Fix: clone AgentAction avec round_num mis a jour

## Performance
- Sans LLM: ~7s / 100 rounds
- Avec LLM (5 agents, 20 rounds): ~50s (4 batch calls x 12s avg)
- Estimation 250 rounds: ~130s (acceptable)
