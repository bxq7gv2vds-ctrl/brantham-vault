# Stratégie Polymarket — Canonique (2026-04-26)

> **Ce document est la source de vérité.** Toute stratégie antérieure
> (`weather-domination-strategy`, plans G1→G2 fondés sur Pangu/EMOS,
> Tier A ITM 0.95+ comme workhorse) est **invalidée** par les findings
> empiriques 2026-04-22 (dedup bug) et 2026-04-25/26 (reverse engineering
> top winners).

## TL;DR

- Edge réel = **METAR observation front-running TTR<2h sur prix 0.05–0.50**, pas NWP forecasting
- Capacity AUM = **$10–20k** plafond. Micro-strat, pas hedge fund scalable
- Notre stratégie actuelle (Tier A ITM 0.95+) capture **1–2% ROI** vs **45–170%** sur le mid-range qu'on ignore
- Pivot : couper le ITM, builder bracket pricer obs-based, paper strict 14j, gate ROI > 0.3%/jour

## Vérités empiriques (non négociables)

### 1. Le dedup bug a invalidé toutes les analyses précédentes
- P&L paper passé inflaté **20×** (sessions, leaderboards, backtests Tier S)
- Vrai P&L = **$980 sur 14j sur $10k bankroll = 0.5% ROI/jour**
- Tier A (NO workhorse) = vrai âme du système, pas Tier S deep_OTM
- Source : [[dedup-bug-p-and-l-inflation]]

### 2. Les vrais winners ne font PAS de NWP sophistiqué
Reverse engineering 3 top winners (`0x05e7` +$69k, `0x42fd` +$28k,
`0x331b` +$27k) → pattern unifié = **METAR observation front-running** :

1. Track temp observée real-time (ASOS 5min cadence, gratuit)
2. À T-2h du settlement, obs + diurnal climato → P(bracket = winner)
3. Achète bracket gagnant à 0.20–0.50 ou lottery NO < 0.05
4. Collecte spread au settlement

**Notre infra Pangu/EMOS est inutile vs ces stratégies.** La compétition
n'est pas modèle vs modèle, c'est **latence observation + bracket
selection rapide**.

Source : [[reverse-engineer-true-winners]]

### 3. Capacity AUM = $10–20k réel
- Volume median Tier S = $749. Notre size $32 = 4.3% volume
- À $100k bankroll : 0% des signaux dans capacity
- Plafond profit weather seul = ~$44k/an
- Source : [[capacity-reality-check]]

- Flag `IS_DUP=1` sur tous les trades duplicates (`market_id + side` collision)

- Critère succès : circuit breaker reset, kill switch ne se trip plus sans cause réelle

**Step 1.2 — Build bracket_pricer_obs.py (P0)**
- Input : `(city, settlement_ts, brackets[])`
- Logic : fetch latest METAR → Tmax-so-far → diurnal climato → P(bracket winner) pour chaque bracket
- Output : `(bracket, p_obs, edge_vs_market)` per bracket
- Critère succès : sur 1 mois historique, AUC ≥ 0.65 sur outcomes connus
- Module : `src/pmhedge/alpha/bracket_pricer_obs.py`

**Step 1.3 — Fix data layer NO tokens (P0)**
- Étendre `polymarket_client.py` pour ingérer `token_id_no` trades
- Backfill 30j historique pour validation
- Critère succès : NO trades visibles dans `trades_onchain.db` pour markets récents

### W2 (2026-05-03 → 2026-05-10) — Wire & paper strict

**Step 2.1 — Wire live executor (P0)**
