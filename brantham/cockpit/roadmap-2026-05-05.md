# Cockpit Brantham — Roadmap 20 étapes (2026-05-05)

Issue de l'audit transverse 2026-05-05 (4 agents parallèles : coverage marché, workflow process, sourcing acquéreurs, qualité produit). Verdict global : plateforme à ~40% du potentiel, vendable single-tenant à 2k€ setup + 200€/mois ; SaaS scalable = 2-3 mois additionnels.

| Axe | Score | Constat |
|---|---|---|
| Coverage marché | 50-60% | 33 AJ scrapés, 8 cassés silencieusement, BODACC absent, cron pas chargé |

## Bloc A — Quick wins infra (jour 1, ~2h)

### 1. Activer le cron daily-scrape ✓ FAIT 2026-05-05

`launchctl load ~/Library/LaunchAgents/com.brantham.daily-scrape.plist`. Job tourne à 8h chaque jour, alimente Supabase via `cockpit.import_scan`. Distinct de `com.brantham.scrape-enrich` qui alimente l'ancien dashboard (toutes les 3h).

**Bug fixé** : `daily_scrape.sh:30` passait `import_scan "$SCAN"` (positionnel) alors que la signature attend `--scan "$SCAN"` (flag). Corrigé.

**Résultats live** : actives 148 → **188** (+40 new), last_scrape_at 2026-04-26 → 2026-05-05, new_24h 0 → 40.

**Blocage TCC restant** : launchd ne peut pas exécuter dans `~/Downloads/` (Operation not permitted). Le job échouera à 8h demain. **Solution** : anticiper l'étape 16 et déplacer `~/Downloads/brantham-pipeline/` → `~/brantham-pipeline/` (path hardcodé aussi dans `cockpit-web/app/api/hunters/stream/route.ts:5`).

### 2. Purger les DLDO passés ✓ FAIT 2026-05-05

Migration `007_purge_dldo_expired.sql` : ajoute colonne `auto_archived_at TIMESTAMPTZ` + index partiel.

Script `cockpit/cron_purge_expired.py` : set `user_status='archive'` + `auto_archived_at=NOW()` quand `date_limite::date < CURRENT_DATE - 7j` ET `user_status='nouveau'` (préserve shortlist/contacted/replied/poubelle/déjà-triés). Idempotent. Flag `--dry-run` et `--days N` dispos.

Intégré à `daily_scrape.sh` après l'import. Tournera donc quotidiennement.

**Résultat** : 263 opps archivées en 1 run. Filtre Vue "⌫ archivées" les retrouve toutes (déjà existant côté UI, juste plus peuplé maintenant).

### 3. Ajouter clés API au `.env` pipeline

- `PAPPERS_API_KEY` (100 tokens/mois gratuit, MCP installé)
- `HUNTER_IO_KEY` (25 emails/mois free)
- Débloque enrichissement contact Soren

### 4. Fix hydration mismatch `Cockpit.tsx:58` ✓ FAIT 2026-05-05

`useState(() => new Date().toLocaleString())` exécuté SSR + CSR avec 1s de drift → React regenerait tout le tree. Fix 2 edits :
- L58 : `useState("")` (SSR/CSR identique = vide)
- L77 : `useEffect` set immédiatement au mount + interval 30s

L'horloge apparaît post-hydratation sans flash. Aucune erreur hydration dans les logs.

## Bloc B — Data quality (semaine 1, ~2j)

### 5. Câbler `enrich_v2.py` post-scrape

Hook fin de `daily_scrape.sh` : SIREN lookup Pappers → NAF + CA + effectif + dirigeant. Cible NAF coverage 0% → 80%. Outil : Pappers MCP ou REST.

### 6. Scraper BODACC API officielle

Source légale data.gouv `bodacc-rcs-a` (RJ/LJ exhaustif). Estimation +30-40% coverage. Outil : `requests` + JSON public.

### 7. Réparer scrapers cassés ✓ FAIT 2026-05-05

État réel après audit live (vs claim audit initial qui était imprécis) :
- **SAJ, Ascagne, Cardon, Maydaymag** : pas cassés (2/18/2/5 opps respectivement)
- **Adjust, KSG, MM AJ, AJ Partenaires** : 0 opps mais sites accessibles → **dormants pas cassés** (petits cabinets régionaux sans annonce active)
- **BVP** : domaine `etude-bpv.fr` 404 → **migré vers `vardon.gemweb.fr`** (cabinet SELARLU BPV / Béatrice Vardon, Paris 75016)

**Bug Gemweb réel trouvé + fixé** dans `_scrape_gemweb_endpoint()` : le scraper récupérait le 1er `t:formdata` du DOM, qui appartenait au form login inline (header `loginformheaderinline.formlogin`) et non au form de recherche. Sur les sites multi-form (Adjust/KSG/MM AJ + autres futurs), le POST renvoyait le formulaire vide au lieu des résultats. Fix : itérer les `t:formdata` en skippant ceux dont le parent form a `action` contenant `loginform`. Futur-proof pour tout nouveau site Gemweb avec login inline.

**BVP réactivé** dans `scraper_aj.py:118` (URL `vardon.gemweb.fr`). Total : 30 sites enabled.

## Bloc C — Génération documents (semaine 2, ~3j)

### 8. Générateur teaser ✓ FAIT 2026-05-05

**Approche** : route Next dédiée `/teaser/[id]` rendue client-side, **éditable inline** (`contentEditable`), `@media print` natif → Cmd+P = PDF propre. Zero dépendance externe (pas de puppeteer, react-pdf, unified, remark).

**Fichiers** :
- `cockpit-web/lib/teaser.ts` — fonction pure `buildTeaser(opp)` : sections (hook, profil, atouts, contexte, recherche, reference) + helpers d'anonymisation et positivisation procédure (RJ/LJ/sauvegarde) selon pattern `teaser-patterns.md`
- `cockpit-web/app/teaser/[id]/page.tsx` — server component qui fetch opp OU deal (fallback : si deal sans opportunity_id, synthétise un Opp shell pour édition manuelle)
- `cockpit-web/components/TeaserDocument.tsx` — A4 layout 210×297mm, header confidentiel, sections éditables, footer mentions légales SAS Brantham + contact Paul, bouton "Imprimer / Exporter PDF" caché à l'impression

**Bouton "Générer teaser"** ajouté dans `OppDrawer.tsx` ET `DealDrawer.tsx` (header), ouvre `/teaser/${id}` dans nouvel onglet.

**Champs manquants** → `[À compléter]` éditables, le user remplit dans le navigateur.

**Magic Form Levallois** : route fonctionne pour le deal (sans opportunity_id), tous les champs en `[À compléter]`. Soren peut éditer, imprimer, envoyer en PDF — débloque la phase 5 du process pour le J-16.

Latence route : ~220ms premier load, 70ms en cache.

### 9. Générateur lettre de mission (LdM)

Bouton "Mandater" dans `DealDrawer` → fusion template `lettre-de-mission.md` + deal.fees + repreneur info → PDF. Table `mandates` (deal_id, type ldm/extension_nda, status, signed_at, pdf_url).

### 10. Générateur courrier extension NDA AJ

Bouton "Demander extension NDA" dans `OppDrawer` → fusion template + opp + AJ contact → PDF + mailto pré-rempli.
