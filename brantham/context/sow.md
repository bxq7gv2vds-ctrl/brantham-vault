# Brantham Partners — Statement of Work (SOW)

> Tous les chantiers a realiser, classes par priorite. L'objectif #1 est clair : **signer le premier mandat repreneur et closer le premier deal.**

---

## Principe directeur

**STOP l'over-engineering. START la vente.**

Chaque chantier ci-dessous est classe selon son impact direct sur le premier deal. Tout ce qui n'aide pas a closer est P2 ou P3.

---

## P0 — BLOQUANT POUR LE PREMIER DEAL

Ces chantiers doivent etre termines AVANT de pouvoir generer du revenu.

### P0.1 — Plaquette de presentation Brantham
**Quoi** : PDF 5-6 slides, design premium, data-driven, effet FOMO
**Pourquoi** : Impossible de contacter des repreneurs sans ca
**Contenu** :
- Slide 1 : Qui on est (positionnement, USP)
- Slide 2 : Le probleme (70K defaillances, marche opaque, opportunities ratees)
- Slide 3 : Notre solution (detection AI, matching, accompagnement complet)
- Slide 4 : Comment ca marche (process simplifie en 4 etapes)
- Slide 5 : Chiffres cles / credibilite (volume de deals detectes, base de donnees, expertise)
- Slide 6 : Contact / CTA
**DA** : Alignee sur le site web (zura-inspired). Brand book a creer en parallele.
**Owner** : Paul (design) + Soren (review)
**Livrable** : PDF + version web
**Status** : ❌ A creer

### P0.2 — Templates outreach
**Quoi** : Tous les supports pour contacter des repreneurs
**Livrables** :

**Owner** : Paul (redaction) + Soren (test terrain)

## P1 — ACCELERATION (des que P0 est fait)

### P1.1 — Lancer l'outreach
**Quoi** : Contacter les 10 premiers repreneurs sur 3-5 deals prioritaires
**Process** :
1. Selectionner 3-5 deals avec deadline > 3 semaines
2. Sourcer 10-20 repreneurs par deal (Pappers + LinkedIn)
3. Envoyer email intro + plaquette + teaser
4. Relance J+3, J+7
5. Si interet → call decouverte → mandat
**Owner** : Soren (outreach) + Paul (support)
**Objectif** : 50-100 emails envoyes, 5-10 calls, 1-2 mandats signes
**Status** : ❌ Bloque par P0

### P1.2 — Definition des roles Paul / Soren
**Quoi** : Formaliser qui fait quoi
**Proposition** :
| Paul | Soren |
|---|---|
| Tech, infra, agents, product | Commercial, outreach, relations AJ |
| Qualification deals | Contact repreneurs |
| Generation teasers | Negociation mandats |
| Maintenance pipeline | Suivi clients |
| Strategie produit | Closing deals |
**Status** : ❌ A discuter et valider

### P1.3 — LinkedIn & presence digitale
**Quoi** :
- Page LinkedIn Brantham Partners
- Premier post (lancement officiel)
- Google Business Profile
- Profils LinkedIn personnels optimises (Paul + Soren)
**Pourquoi** : Credibilite quand les repreneurs googlelent "Brantham Partners"
**Owner** : Soren
**Status** : ❌ A creer

### P1.4 — CRM minimal
**Quoi** : Tracker les contacts repreneurs, les deals, les relances
**Options** : Notion, Airtable, ou directement dans le dashboard React
**Pourquoi** : Sans tracking, on perd le fil des contacts
**Owner** : Paul (setup) + Soren (utilisation quotidienne)
**Status** : ❌ A creer

## P2 — OPTIMISATION (une fois les premiers deals en cours)

### P2.1 — Automatisation pipeline agents
**Quoi** : Mettre les agents en production autonome (scoring auto → teaser auto → buyer mapping auto)
**Pourquoi** : Scale sans ajouter de temps humain par deal
**Prerequis** : Au moins 2-3 deals traites manuellement pour calibrer
**Owner** : Paul
**Status** : Agents construits, pas en production autonome

### P2.2 — Dashboard interne V2
**Quoi** : Refactor du dashboard React pour usage quotidien reel
**Priorites** :
- Pipeline kanban fonctionnel (drag-drop deals)
- Fiche deal complete (teaser + buyers + contacts + status)
- CRM integre (contacts repreneurs + historique interactions)
- Notifications (deadlines, relances)
**Owner** : Paul
**Status** : ~60% construit, pas utilise en vrai

**Quoi** : Continuer le travail SEO pour les leads entrants
- Submit nouvelles pages GSC
- Case studies anonymises (3-5)

- Blog posts / insights marche
**Owner** : Paul
**Status** : Site lance, pas de leads encore

**Quoi** : contact@brantham-partners.com (ou equivalent)
**Pourquoi** : Credibilite outreach. Pas d'email @gmail.
**Owner** : Paul
**Status** : ❌ A verifier

## Anti-patterns a eviter

1. **"Je vais juste ameliorer X avant de lancer"** → NON. Lance avec ce que tu as.
2. **"Les agents ne sont pas encore parfaits"** → On s'en fout. Les premiers deals sont manuels.
3. **"Il faut d'abord finir le dashboard"** → Le dashboard est pour toi, pas pour le client. Un PDF suffit.
4. **"MiroFish a besoin de calibration"** → MiroFish est P3. Le revenu est P0.
5. **"Je vais scraper plus de sources"** → Tu as deja 468 opportunites. Le probleme c'est pas la detection, c'est l'outreach.
