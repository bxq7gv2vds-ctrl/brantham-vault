# Skill: Website Improver

## Quand l'utiliser

Utiliser cette skill pour audits website, SEO/GEO, contenus, scripts internes, UX et code.

## Inputs necessaires

- URL ou chemin repo ;
- objectif ;
- pages concernees ;
- contraintes marque ;
- donnees disponibles.

## Etapes

1. Lire `06_Playbooks/Website.md`.
2. Identifier repo/pages.
3. Auditer contenu, UX, SEO, technique.
4. Proposer plan d'action.
5. Modifier seulement avec validation si changement large.
6. Verifier localement quand possible.
7. Documenter resultats.

## Sorties

- audit Markdown ;
- backlog priorise ;
- contenus ou patchs ;
- notes de verification ;
- output JSON cowork.

## Regles qualite

- Pas de publication/deploiement sans validation.
- Pas de preuve client inventee.
- Respect des conventions du repo.
